O sistema em questão deve ser capaz de organizar algumas atividades de uma ONG. As atividades se referem à coordenação de Campanhas e Doações.

Uma Campanha remete à uma meta a ser buscada em um tempo para auxiliar uma determinada Entidade. As Campanhas podem ser para arrecadação de alimentos, roupas, itens de limpeza, itens para construção dentre outros.

Exemplos de Campanha:

Campanha para arrecadar alimentos perecíveis
Necessita-se dos seguintes Itens: 
100kg de arroz
 50kg de
feijão
 20kg de macarrão
Os Itens serão doados para a Entidade: Casa de Auxílio aos Idosos ABC
Prazo de Vencimento: 30/10/2022

Após o lançamento da Campanha, a ONG se abre para receber Doações, realizadas por Pessoas que desejam ajudar.

Exemplos de Doação:

Pessoa: Geraldo
Item: 10kg de arroz

Pessoa: SH Comércio
Item: 10 pacotes de papel higiênico

O que o sistema deve fazer:
As Campanhas, as Doações, assim como todos os dados ao redor
desses dois pontos devem ser cadastrados pelo sistema. O Usuário deve informar
via terminal todos esses dados
O sistema deve ser capaz de informar o status de andamento
de uma Campanha a partir das Doações que já foram recebidas. Nessa simulação, uma Doação usada para uma Campanha não pode ser reaproveitada para outra Campanha. As Campanhas devem ser priorizadas de acordo com o prazo de vencimento.
Quando uma Campanha tiver sua meta de Doações atendidas, o Usuário poderá fechar a Campanha. Neste caso, as Doações usadas nessa Campanha
saem do estoque disponível.
Quando uma Campanha vencer o prazo, e não tiver a meta
atendida, o usuário poderá: estender o prazo, mudar os itens da campanha, ou
fechar a campanha.
O sistema deve organizar as funcionalidades na forma de um menu
